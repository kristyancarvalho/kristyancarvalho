# Profile contribution stats

The contribution widget is generated by `scripts/generate-readme-assets.go` and written to `assets/readme/contributions.svg`.

## Token

Set the repository secret `PROFILE_STATS_TOKEN` to include private/internal contribution counts.

Recommended token setup:

- Classic personal access token: `read:user` for private contribution counts.
- Add `repo` only if private pull request line estimates should include private repositories.
- Fine-grained personal access token: select only the needed repositories and grant read-only metadata and pull request access. If private contribution counts do not appear, use a classic token with `read:user`, because GitHub exposes private contribution counts through that scope.

The workflow never prints the token. The generator also avoids logging raw GraphQL responses.

## Privacy model

The widget only renders aggregate numbers and anonymous time-series bars. It does not request or render repository names, branch names, commit messages, file paths, URLs, PR titles, or raw contribution records for the contribution widget.

## Line-count limitation

GitHub GraphQL exposes aggregate contribution counts directly, but it does not expose an exact lifetime "lines of code touched" total through `ContributionsCollection`. The widget therefore uses the best safe approximation available from the API: it sums `additions` and `deletions` from pull requests authored by the profile user and visible to `PROFILE_STATS_TOKEN`, without selecting private repository metadata.

This means direct commits, non-PR work, inaccessible repositories, and search result caps can make line totals lower than the real total.
